# 用户使用指南

这份文档只讲怎么使用，不讲代码细节。你可以把它当成每天打开工具前的操作说明。

## 1. 启动方式

推荐方式：双击项目根目录的：

```text
启动投资分析.bat
```

正常情况下会自动打开浏览器。如果没有自动打开，可以手动访问：

```text
http://localhost:8501
```

如果窗口闪退，先看 [常见问题排查](TROUBLESHOOTING.md)。

## 2. 第一次配置

第一次使用需要安装依赖并创建 `.env`：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
notepad .env
```

`.env` 里至少需要一个 AI provider 的 key。推荐先用 Gemini：

```text
GEMINI_API_KEY=你的GeminiKey
```

如果你使用本地代理访问 Gemini，可以加：

```text
AI_PROXY=http://127.0.0.1:7890
```

## 3. 输入标的

网页里可以直接输入代码：

```text
600519
510300
LC2609
```

也可以明确指定资产类型：

```text
stock:600519
etf:510300
futures:LC2609
```

如果自动识别不准，使用显式前缀最稳。

## 4. 页面选项怎么选

- 资产类型：默认“自动识别”。遇到期货或 ETF 识别不准时手动选择。
- AI provider：默认 Gemini。Gemini 不通时可以试 DeepSeek。
- 报告深度：默认“专业详细”。如果想更快，可以选较短报告。
- 天数：决定拉取多少历史 K 线。短线看 120 天基本够用，想看中期结构可以调到 250 天。
- 生成 HTML 报告：建议开启，方便保存和复盘。

## 5. 怎么阅读报告

阅读顺序建议如下：

1. 先看“一句话结论”：快速知道当前偏多、偏空、震荡还是观望。
2. 再看“当前趋势结构”：判断价格是在上升趋势、下跌趋势还是修复阶段。
3. 看“支撑/压力”：这些位置决定计划是否还能成立。
4. 看“风险收益比”：优秀投资者不会只问会不会涨，还会先问亏多少、赚多少、值不值得。
5. 看“仓位建议”：新手重点看稳健方案，不要只看激进方案。
6. 看“看错条件”：价格走到这个条件时，原判断就要重新评估。
7. 最后看“新闻与来源链接”：自己点开链接复核新闻是否真实、是否过期、是否强相关。

网页还会显示“量化因子与历史相似信号”。这个区域不是 AI 写出来的，而是程序本地计算的轻量量化底稿，重点看：

- 当前量化信号：例如趋势多头、趋势空头、弱势超跌、震荡中性。
- 因子综合分：趋势、动量、波动、量能、位置等因子的综合结果。
- 历史相似信号：过去出现类似信号后，未来 1 日、3 日、5 日的胜率和收益分布。
- 策略型回测：按下一交易日开盘入场、ATR 止损止盈、最多持有 5 日，并扣除简化交易成本后的模拟表现。
- 样本数：样本太少时，结论只能弱参考。

注意：当前策略回测已经包含简化交易成本，但仍不含真实滑点、盘口冲击、样本外验证和组合仓位管理，不能当成确定预测。

新版网页和 HTML 报告会直接解释这些指标：

- 趋势因子：看价格和 MA20/MA60 的关系，判断趋势是否顺。
- 动量因子：看 RSI、MACD、近 5 日和 20 日涨跌，判断上涨/下跌动力。
- 波动风险因子：看最近波动率和单日最大跌幅，判断持仓过程是否容易大起大落。
- 量能因子：看成交量是否放大，以及放量时是上涨还是下跌。
- 位置/盈亏比因子：看当前价离支撑和压力有多近，判断追高或低吸是否划算。
- 胜率：历史类似信号里，按当前方向赚钱的比例。
- 平均收益：历史样本的平均表现，要和最大亏损一起看。
- 盈亏比：盈利样本总收益除以亏损样本总亏损，大于 1 才比较健康。
- 最大回撤：模拟策略曾经从高点跌到低点的最大亏损幅度。

如果你看不懂某个量化词，优先看报告里的“指标词典”和“怎么看”列。

报告里的“关键价位分层”分三类：

- 短线观察位：适合今天/明天使用，例如日内高低点、上一收盘价、MA5/MA10。
- 波段关键位：适合判断一段趋势是否还成立，例如 MA20、布林带、20 日高低点。
- 极端风险位：适合评估最坏情景，例如 MA60、60 日高低点，不应直接当成短线止损。

网页生成报告后，会在 AI 报告下方出现“继续追问”区域。你可以继续问：

```text
明天怎么操作
为什么这么判断
风险在哪里
如果我已经持仓怎么办
```

这些追问会基于当前这份数据底稿继续分析。如果想换标的，在侧边栏输入新代码并重新点击“生成分析”。

如果 AI provider 使用 Gemini，追问默认会启用 Gemini 自己的 Google Search grounding。也就是说，当你问“去搜索仓单”“查一下库存”“核实最新现货价格”“找相关新闻”时，AI 会自己调用搜索工具，并在回答末尾给出搜索词和来源链接。这个能力可以在 `.env` 中关闭：

```text
GEMINI_SEARCH_GROUNDING=false
```

如果追问里包含“走势图”“画图”“今天走势”“盘中”“预测今日”等词，网页会在 AI 回答下方额外生成一张“今日走势情景预测图”。这张图会展示：

- 已发生的盘中走势，如果当前数据里有分钟线。
- 基准预测路径。
- 乐观路径。
- 悲观路径。
- 当前价、第一支撑和第一压力。

这张图是基于当前数据底稿生成的情景推演，不是确定走势。

## 6. 股票、ETF、期货的报告重点不同

股票报告重点：

- 盈利能力
- 成长能力
- 现金流质量
- 估值水平
- 财务安全
- 行业与竞争
- 技术面与市场情绪

ETF 报告重点：

- 跟踪指数方向
- 成交额和流动性
- 板块估值与拥挤度
- IOPV 和折溢价
- 最新份额和市值
- 最近一期前十大/前十五大持仓
- 行业配置和集中度
- 成分股风险
- 技术趋势

期货报告重点：

- 趋势和动量
- 现货价格和基差
- 持仓排名变化
- 成交量和波动率
- 产业供需新闻
- 杠杆和止损风险

## 7. HTML 报告在哪里

报告默认保存在：

```text
reports/
```

文件名类似：

```text
LC2609_20260528_155609.html
```

可以用浏览器直接打开，也可以发给别人看。

## 8. CLI 备用命令

网页无法启动时，可以用命令行：

```powershell
.\.venv\Scripts\python main.py futures:LC2609 --ai-provider gemini
```

不调用 AI，只检查数据是否能取到：

```powershell
.\.venv\Scripts\python main.py futures:LC2609 --no-ai
```

## 9. 使用时的基本原则

- 不要因为 AI 说“偏多”就满仓。
- 新闻一定要看来源和时间。
- 期货必须先确定止损，再考虑开仓。
- 数据异常时宁可不交易。
- 报告是决策辅助，不是自动交易指令。
