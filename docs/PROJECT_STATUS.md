# 项目当前状态

这份文档说明项目“现在已经做到什么、还没做到什么、下一步该怎么做”。它适合在读代码前先看，避免对项目能力产生误解。

## 1. 当前定位

当前版本是：

```text
AI-first 投资分析助手
```

它不是完整的专业量化系统，也不是自动交易系统。它的核心价值是：

- 自动收集公开行情和新闻。
- 整理技术指标、期货基差、持仓排名等底稿。
- 让 AI 按较专业的投资框架输出报告。
- 生成图文 HTML 报告，方便复盘。

## 2. 已完成能力

- 本地网页入口：`web_app.py`。
- 双击启动脚本：`启动投资分析.bat`。
- CLI 备用入口：`main.py`。
- 股票、ETF、期货资产类型识别。
- 显式前缀输入：`stock:`、`etf:`、`futures:`。
- AKShare 行情、K 线、新闻、部分基本面数据。
- 免费网页新闻搜索。
- Gemini/DeepSeek/OpenAI provider 切换。
- Gemini 官方 REST 调用和代理配置。
- HTML 图文报告。
- 新闻来源链接展示。
- 期货现货/基差、结算、持仓排名增强。
- 专业投研上下文：新闻相关性、基差解释、持仓聚合、风险收益比。

## 3. 当前不足

### 不是严格量化模型

当前报告主要是“数据底稿 + AI 综合判断”。它还没有系统验证：

- 某个信号过去出现多少次。
- 未来 1/3/5 日胜率是多少。
- 平均收益和最大亏损是多少。
- 信号在不同市场阶段是否失效。

### 免费数据源有局限

AKShare 和公开网页新闻适合学习和研究，但不能保证像专业终端一样稳定完整。

### 新闻质量需要人工复核

程序会显示链接，但新闻是否强相关、是否过期、是否标题夸张，仍然需要人工判断。

### 基本面数据不一定完整

股票财务和估值数据依赖上游接口。取不到字段时，AI 必须说明缺口。

### 期货还缺少更多产业数据

例如库存、仓单、期限结构、上下游价格、交易所公告等还没有系统化接入。

## 4. 使用边界

适合：

- 学习 Python 项目结构。
- 生成投研初稿。
- 对标的做快速多维度检查。
- 保存报告用于复盘。
- 辅助发现风险点和观察位。

不适合：

- 直接自动下单。
- 替代交易软件行情。
- 替代专业投顾意见。
- 单凭 AI 结论重仓交易。

## 5. 近期最重要的升级方向

下一阶段建议做“轻量专业量化版”：

1. 新增量化因子模块。
2. 新增历史回测模块。
3. 输出当前信号的历史胜率和收益分布。
4. 让 AI 必须引用回测数据。
5. 网页展示“量化因子与回测”区块。

这样项目会从：

```text
AI 判断
```

升级为：

```text
历史验证 + 当前信号 + AI 解读
```

## 6. 今天读代码的目标

你今天不需要完全掌握所有细节。最重要的是理解：

- 用户输入如何进入系统。
- 系统如何判断资产类型。
- 数据从哪里来。
- 指标在哪里计算。
- AI prompt 在哪里组织。
- HTML 报告在哪里生成。
- 以后要改哪个功能应该找哪个文件。
