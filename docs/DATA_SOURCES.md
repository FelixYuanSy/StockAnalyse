# 数据源说明

本项目主要使用 AKShare 和公开网页新闻。优点是成本低、部署简单；缺点是稳定性、实时性和字段完整度无法和 Wind、Choice、同花顺 iFinD、Bloomberg 等专业终端相比。

所以报告里所有数据都应该理解为“研究辅助底稿”，不能当成交易所直连行情。

## 1. 股票数据

股票使用的主要数据：

| 数据类型 | 来源接口 | 用途 |
| --- | --- | --- |
| 实时行情 | `stock_zh_a_spot_em` | 当前价、涨跌幅、成交量、换手率 |
| 历史 K 线 | `stock_zh_a_hist` | 日线走势、均线、RSI、MACD |
| 备用日线 | `stock_zh_a_daily` | 主接口失败时兜底 |
| 个股新闻 | `stock_news_em` | 公司相关新闻 |
| 公司信息 | `stock_individual_info_em` | 公司基础资料 |
| 财务摘要 | `stock_financial_abstract` | 盈利能力、成长能力等 |
| 财务指标 | `stock_financial_analysis_indicator` | ROE、ROA、利润率等 |
| 历史估值 | `stock_zh_valuation_baidu` | PE/PB 历史参考 |
| 同行估值 | `stock_zh_valuation_comparison_em` | 同行业对比 |

股票报告会尽量按 100 分框架组织：

- 盈利能力
- 成长能力
- 现金流质量
- 估值水平
- 财务安全
- 行业与竞争
- 技术面与市场情绪

如果某些财务字段取不到，AI 必须说明数据缺口，不能假装已经分析完整。

## 2. ETF 数据

ETF 使用的主要数据：

| 数据类型 | 来源接口 | 用途 |
| --- | --- | --- |
| 实时行情 | `fund_etf_spot_em` | 当前价、涨跌幅、成交额 |
| 历史 K 线 | `fund_etf_hist_em` | 趋势、波动、技术指标 |
| 备用 K 线 | `fund_etf_hist_sina` | 主接口失败时兜底 |
| IOPV/折溢价 | `fund_etf_spot_em` | IOPV实时估值、基金折价率、最新份额、市值 |
| 持仓明细 | `fund_portfolio_hold_em` | 最近一期股票持仓和集中度 |
| 行业配置 | `fund_portfolio_industry_allocation_em` | 最近一期行业占比 |
| 净值历史 | `fund_etf_fund_info_em` | 单位净值、累计净值、日增长率 |
| 市场新闻 | `stock_info_global_em` | 市场背景 |
| 网页新闻 | `news_searcher.py` | 指数、板块、宏观新闻补充 |

ETF 报告重点不是公司财务，而是：

- 跟踪指数方向。
- 成交额和流动性。
- 行业或宽基指数估值。
- 成分股集中度。
- IOPV、折溢价、最新份额和市值。
- 最近一期持仓和行业配置。
- 技术趋势和资金情绪。

## 3. 期货数据

期货使用的主要数据：

| 数据类型 | 来源接口 | 用途 |
| --- | --- | --- |
| 实时行情 | `futures_zh_spot` | 当前价、涨跌幅、成交量 |
| 历史日线 | `futures_zh_daily_sina` | 趋势、波动、技术指标 |
| 主连备用 | `futures_main_sina` | 单合约日线不可用时兜底 |
| 分钟线 | `futures_zh_minute_sina` | 短线观察 |
| 产业新闻 | `futures_news_shmet` | 商品相关资讯 |
| 网页新闻 | `news_searcher.py` | 公开网页新闻补充 |
| 现货/基差 | `futures_spot_price` | 期货相对现货贵还是便宜 |
| 结算参数 | `futures_settle` | 结算价、保证金等参考 |
| 持仓排名 | `futures_hold_pos_sina` | 成交、多单、空单排名和变化 |

期货报告重点：

- 趋势方向。
- 现货价格和基差。
- 持仓排名变化。
- 成交量和波动率。
- 产业供需新闻。
- 杠杆和止损风险。

期货不适合套用股票财务评分，因为期货交易的是合约价格，不是公司股权。

## 4. 网页新闻搜索

`stock_analyse/news_searcher.py` 会根据标的自动生成关键词。例如 `LC2609` 会搜索：

- `LC2609 碳酸锂`
- `碳酸锂 期货 最新`
- `碳酸锂 现货价格 基差`
- `碳酸锂 库存 仓单 供需`
- `SMM 碳酸锂`
- `生意社 碳酸锂`
- `广期所 碳酸锂`

抓取到的新闻会保留：

- 标题
- 来源
- 发布时间
- 摘要
- 原文链接

HTML 报告和网页页面会显示链接，方便你手动复核。

## 5. 新闻可信度原则

AI 报告不应该把所有新闻同等看待。当前提示词要求 AI 区分：

- 强相关新闻：直接影响标的供需、业绩、政策或价格。
- 弱相关新闻：行业背景或间接影响。
- 市场背景新闻：宏观、海外、汇率、风险偏好等。
- 过期新闻：只能作为背景，不能当作最新驱动。

如果新闻没有来源、没有时间、无法打开链接，应降低权重。

## 6. 数据准确性能否保证

不能保证 100% 准确。原因：

- AKShare 是公开数据封装，上游接口可能变化。
- 免费网页新闻可能有延迟、重复、转发或标题党。
- 期货实时行情和日线可能来自不同数据源，时间口径不一定一致。
- 现货价格和基差可能不是交易所官方实时数据。

因此专业使用流程应该是：

1. 用本工具发现线索。
2. 点开新闻链接复核原文。
3. 对关键行情到交易软件或交易所官网二次确认。
4. 真实交易前确认止损、仓位和流动性。

## 7. 下一阶段数据增强方向

向专业量化模型升级时，数据层需要补强：

- 本地缓存历史数据，减少接口波动。
- 保存每次分析时的数据快照，方便复盘。
- 增加期货仓单、库存、期限结构数据。
- 增加指数估值、行业景气数据。
- 给每个数据字段记录来源、时间和可用性。
