# 量化升级路线

这份文档描述下一阶段如何把当前 AI 投研助手升级为“轻量专业量化模型”。重点不是一开始就做复杂机器学习，而是先建立可验证的因子和回测体系。

## 1. 为什么要升级

当前工具的问题是：AI 能解释数据，但它没有自动证明某个信号在历史上是否真的有效。

专业量化更关心：

- 信号过去出现过多少次。
- 出现后未来 1/3/5 日涨跌概率如何。
- 平均收益、最大亏损、盈亏比如何。
- 是否只在某些市场环境有效。
- 当前信号是否值得承担风险。

所以升级方向应该是：

```text
行情数据
  -> 因子计算
  -> 历史回测
  -> 当前信号评分
  -> AI 解释
```

## 2. 第一阶段：因子模块

新增文件建议：

```text
stock_analyse/factors.py
```

当前实现文件：

```text
stock_analyse/quant.py
```

已经实现：

- 趋势因子
- 动量因子
- 波动风险因子
- 量能因子
- 位置/盈亏比因子
- ETF 流动性/折溢价因子
- ETF 持仓集中度因子
- 期货基差因子
- 期货持仓因子

先实现这些通用因子：

| 因子 | 含义 |
| --- | --- |
| trend_factor | 趋势方向，基于均线、多空排列、均线斜率 |
| momentum_factor | 动量强弱，基于 RSI、MACD、近 N 日涨跌幅 |
| volatility_factor | 波动风险，基于 ATR、历史波动率、最大回撤 |
| volume_factor | 成交量异常，基于放量、缩量、量价配合 |
| level_factor | 价格离支撑/压力的位置 |

期货额外因子：

| 因子 | 含义 |
| --- | --- |
| basis_factor | 期货相对现货升贴水 |
| position_factor | 持仓排名和多空变化 |
| term_structure_factor | 期限结构，近远月强弱 |
| inventory_factor | 库存/仓单变化，后续接入 |

股票额外因子：

| 因子 | 含义 |
| --- | --- |
| quality_factor | 盈利质量、ROE、现金流 |
| growth_factor | 收入和利润增长 |
| valuation_factor | PE/PB/PEG/股息率 |
| safety_factor | 负债、现金短债比、利息保障 |

ETF 额外因子：

| 因子 | 含义 |
| --- | --- |
| index_trend_factor | 跟踪指数趋势 |
| liquidity_factor | 成交额和价差 |
| concentration_factor | 成分集中风险 |

## 3. 第二阶段：回测模块

新增文件建议：

```text
stock_analyse/backtest.py
```

当前实现先合并在：

```text
stock_analyse/quant.py
```

已经实现轻量事件型回测：

- 自动识别当前信号：趋势多头、趋势空头、弱势超跌、高位过热、放量偏强、震荡中性。
- 在历史 K 线中寻找同类信号。
- 统计未来 1 日、3 日、5 日表现。
- 输出样本数、胜率、平均收益、中位数收益、最大收益、最大亏损、方向收益和盈亏比。

先做事件型回测，不急着做复杂策略。

例子：

```text
当 RSI < 30 且价格低于 MA20 时，
统计未来 1/3/5 日收益。
```

输出指标：

- 样本数
- 上涨胜率
- 平均收益
- 中位数收益
- 最大收益
- 最大亏损
- 盈亏比
- 最大回撤

这样 AI 报告可以写：

```text
过去 120 个交易日内，类似信号出现 8 次。
未来 3 日上涨胜率 37.5%，平均收益 -1.2%，最大亏损 -4.8%。
因此当前不适合直接抄底，除非重新站上关键压力位。
```

## 4. 第三阶段：当前信号评分

新增文件建议：

```text
stock_analyse/signal_engine.py
```

输出当前市场状态：

- 趋势多头
- 趋势空头
- 震荡
- 超跌修复
- 高位过热
- 放量突破
- 缩量反抽

同时输出：

- 量化评分
- 信号置信度
- 适合周期
- 适合仓位
- 失效条件

当前已在 `quant_context` 中输出：

- `signal`：当前量化信号。
- `factor_score`：因子综合分。
- `confidence`：量化置信度。
- `factors`：各因子分数和证据。
- `event_backtest`：历史相似信号统计。

## 5. 第四阶段：AI 报告升级

`ai_advisor.py` 的 prompt 要求 AI 必须引用：

- 当前因子分数。
- 回测样本数。
- 胜率。
- 平均收益。
- 最大亏损。
- 风险收益比。

AI 不允许只写“我认为会涨/会跌”。必须写：

```text
我的判断依据是：趋势因子、动量因子、基差因子、历史回测结果和新闻驱动。
```

当前 `ai_advisor.py` 已要求：如果 `analysis_payload.fundamental_data.quant_context.available = true`，AI 必须增加“量化因子与历史相似信号”章节，并引用因子分、信号类型、1/3/5 日回测表现。

## 6. 第五阶段：网页展示升级

`web_app.py` 增加：

- 因子雷达图。
- 回测统计表。
- 当前信号卡片。
- 多周期预测：1 日、3 日、5 日。
- 样本数不足提示。

HTML 报告增加：

- “量化因子”区块。
- “历史相似信号回测”区块。
- “AI 如何使用回测结果”说明。

当前 `web_app.py` 已增加“量化因子与历史相似信号”展示区，会显示：

- 当前量化信号。
- 因子综合分。
- 量化置信度。
- 因子表。
- 1/3/5 日相似信号回测表。

## 7. 第一版不要做什么

第一版先不要做：

- 自动交易。
- 高频 tick 回测。
- 复杂机器学习模型。
- 需要大量收费数据的功能。
- 无法解释的黑箱预测。

原因很简单：先把可解释、可验证的基础打稳。

## 8. 推荐实施顺序

1. 新建 `factors.py`，先做趋势、动量、波动、成交量因子。
2. 新建 `backtest.py`，先做未来 1/3/5 日事件回测。
3. 把回测结果放入 AI payload。
4. 修改 prompt，让 AI 必须引用回测数据。
5. 网页展示因子和回测结果。
6. 再扩展期货基差、持仓和库存因子。

## 9. 判断是否升级成功

升级成功的标准不是“报告更长”，而是报告能回答：

- 当前信号过去出现过几次？
- 历史胜率是多少？
- 平均收益是多少？
- 最大亏损是多少？
- 如果买错，在哪里退出？
- 这笔交易值不值得承担风险？

当这些问题能自动回答时，项目才真正开始接近专业量化模型。
